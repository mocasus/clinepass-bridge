Quick Start

1. Clone & install

git clone https://github.com/mocasus/clinepass-bridge.git
cd clinepass-bridge
npm install

2. Generate API key

npm run genkey
# → sk-cpb-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

3. Configure .env

cp .env.example .env
# edit .env:
API_KEYS=sk-cpb-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx

4. Start server

npm run dev
# → http://127.0.0.1:8787

5. Verify

curl -s http://127.0.0.1:8787/v1/chat/completions \
  -H "Authorization: Bearer *** \
  -H "Content-Type: application/json" \
  -d '{"model":"kimi-k3","messages":[{"role":"user","content":"hi"}]}'

✅ Got "choices" in the response? Done. Point any tool at http://127.0.0.1:8787/v1.

Python

from openai import OpenAI
client = OpenAI(base_url="http://127.0.0.1:8787/v1", api_key="sk-cpb-…")
print(client.chat.completions.create(
    model="kimi-k3",
    messages=[{"role": "user", "content": "hello"}]
).choices[0].message.content)

Deploy

Docker

docker build -t clinepass-bridge .
docker run -d -p 8787:8080 --name clinepass-bridge \
  -e HOST=0.0.0.0 -e PORT=8080 \
  -e API_KEYS=*** \
  -e CLINE_REFRESH_TOKEN=*** \
  -v clinepass-cache:/app/.cache \
  clinepass-bridge

Fly.io

fly launch --no-deploy
fly secrets set \
  API_KEYS=*** \
  CLINE_REFRESH_TOKEN=*** deploy

⚠ Avoid serverless (Vercel/Netlify) — timeouts kill LLM streaming.

Troubleshooting

SymptomFix
apiKeysConfigured: 0.env not loaded. Make sure API_KEYS is set.
No refresh token availableSign in with Cline CLI once, or set CLINE_REFRESH_TOKEN.
Streaming cuts offDon't use serverless. Use a VM (Railway, Fly.io, VPS).
Port already in useChange PORT or kill existing: lsof -ti:8787 | xargs kill
401/403 from upstreamToken expired. Sign in with Cline CLI again.
connection refusedBridge not running. npm run dev first.
