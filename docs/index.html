Features

Zero-config. Auto-refresh. Always fresh.

Drop it in, generate an API key, and forget about token management.

🔑

Your own API keys

Generate sk-cpb-… keys for each of your tools. Multiple keys supported with timing-safe validation.

🔄

Auto-refresh tokens

Reads directly from the Cline CLI credential store. When tokens expire, refreshes via WorkOS and writes back — zero babysitting.

🌊

Streaming & non-streaming

Full SSE streaming passthrough for real-time responses. Non-streaming with automatic response envelope unwrapping.

🏷️

Smart model aliases

Use short names like kimi-k3 or glm-5.2. The bridge resolves them to cline-pass/kimi-k3 automatically.

🔁

401 auto-retry

When the upstream rejects a stale token, the bridge force-refreshes and retries — transparent to your client.

☁️

Deploy anywhere

One codebase for local dev and cloud. Ships with Dockerfile, Fly.io config, and Railway-ready setup.

Quick Start

Up in 60 seconds.

You need the Cline CLI signed in once. The bridge reads your tokens from there.

1. Clone & install
git clone https://github.com/mocasus/clinepass-bridge.git
cd clinepass-bridge
npm install
2. Generate an API key
npm run genkey
# → sk-cpb-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
3. Configure .env
cp .env.example .env
# Add your key:
API_KEYS=sk-cpb-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
4. Start
npm run dev
# → http://127.0.0.1:8787
5. Try it
curl http://127.0.0.1:8787/v1/chat/completions \
  -H "Authorization: Bearer sk-cpb-…" \
  -H "Content-Type: application/json" \
  -d '{"model":"kimi-k3","messages":[{"role":"user","content":"hello"}]}'

✅ Got a JSON reply with "choices"? You're live. Point any OpenAI-compatible client at http://127.0.0.1:8787/v1.

Deploy

Local or cloud. Same code.

Only the environment variables change between dev and production.

🐳

Docker

Multi-stage build. 256 MB. Runs on any VPS.

🪰

Fly.io

Pre-configured fly.toml. Free-tier friendly with auto-stop.

🚂

Railway

Auto-detects Dockerfile. One-click deploy.

Docker (any VPS)
docker build -t clinepass-bridge .
docker run -d -p 8787:8080 --name clinepass-bridge \
  -e HOST=0.0.0.0 -e PORT=8080 \
  -e API_KEYS=sk-cpb-… \
  -e CLINE_REFRESH_TOKEN=… \
  -v clinepass-cache:/app/.cache \
  clinepass-bridge
Fly.io
fly launch --no-deploy
fly secrets set \
  API_KEYS=sk-cpb-… \
  CLINE_REFRESH_TOKEN=…
fly deploy

⚠️ Avoid serverless (Vercel/Netlify) — their request timeouts kill LLM streaming. Use a VM-style host or run locally.

Troubleshooting

Common issues.

🔴 "apiKeysConfigured: 0"

Your .env wasn't loaded. Make sure it's in the project root and API_KEYS is set (not commented out).

🔴 "No refresh token available"

Sign in with the Cline CLI once, or set CLINE_REFRESH_TOKEN in your .env.

🔴 Streaming cuts off

Don't use serverless platforms (Vercel/Netlify). Their timeouts kill long-running SSE connections.

🔴 Port already in use

Change PORT in .env or kill the existing process: lsof -ti:8787 | xargs kill
