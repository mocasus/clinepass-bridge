clinepass-bridge

An OpenAI-compatible API bridge that lets you use your Cline Pass subscription from any tool — Cursor, Continue.dev, Aider, LibreChat, or the openai SDK.

Introduction

The bridge sits between your favorite AI tool and api.cline.bot, translating standard OpenAI requests into Cline Pass calls. It handles authentication, token refresh, and streaming — so you never think about credentials.

architecture.txt 
[ Your tool ]
   │  OpenAI API + sk-cpb-…
   ▼
[ clinepass-bridge ]
   │  reads ~/.cline/data/settings/providers.json
   │  auto-refreshes WorkOS tokens
   ▼
[ api.cline.bot ]
   kimi-k3 · glm-5.2 · deepseek-v4-pro · +8 more
ℹ️
How it works: the bridge reads your local Cline CLI credential store, so as long as you use the Cline CLI or VS Code extension, tokens stay fresh automatically.

Quick Start

Get up and running in under a minute.

Clone & install
bash 
git clone https://github.com/mocasus/clinepass-bridge.git
cd clinepass-bridge
npm install
Generate your API key
bash 
npm run genkey
# → sk-cpb...xxxx
Configure .env
.env 
API_KEYS=sk-cpb...xxxx
Start the bridge
bash 
npm run dev
# → http://127.0.0.1:8787
Verify it works
bash 
curl -s http://127.0.0.1:8787/v1/chat/completions \
  -H "Authorization: Bearer sk-cpb-…" \
  -H "Content-Type: application/json" \
  -d '{"model":"kimi-k3","messages":[{"role":"user","content":"hi"}]}'
✅
Got a JSON reply with "choices"? You're live. Point any OpenAI-compatible tool at http://127.0.0.1:8787/v1.

Authentication

The bridge resolves a valid Cline access token in this order:

  1. In-memory token (fastest)
  2. Bridge-local cache file (.cache/tokens.json)
  3. Cline CLI credential store (~/.cline/data/settings/providers.json)
  4. Environment override (CLINE_ACCESS_TOKEN)
  5. WorkOS refresh_token grant (single-flight, persisted)
🔑
When a token is refreshed, it's written back to both the bridge cache and the Cline CLI store, keeping CLI + bridge in sync.

SDK Examples

from openai import OpenAI

client = OpenAI(
    base_url="http://127.0.0.1:8787/v1",
    api_key="sk-cpb-…"
)

res = client.chat.completions.create(
    model="kimi-k3",
    messages=[{"role": "user", "content": "hello"}]
)
print(res.choices[0].message.content)
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "http://127.0.0.1:8787/v1",
  apiKey: "sk-cpb-…",
});

const res = await client.chat.completions.create({
  model: "glm-5.2",
  messages: [{ role: "user", content: "hello" }],
});
console.log(res.choices[0].message.content);
curl -s http://127.0.0.1:8787/v1/chat/completions \

  -H "Content-Type: application/json" \
  -d '{
    "model": "kimi-k3",
    "messages": [{"role":"user","content":"hello"}]
  }'

Streaming

Add "stream": true to any request — the bridge passes SSE through verbatim.

streaming.py 
from openai import OpenAI

client = OpenAI(base_url="http://127.0.0.1:8787/v1", api_key="sk-cpb-…")

stream = client.chat.completions.create(
    model="kimi-k3",
    messages=[{"role": "user", "content": "count to 10"}],
    stream=True,
)

for chunk in stream:
    print(chunk.choices[0].delta.content or "", end="")
⚠️
Serverless warning: avoid Vercel / Netlify for streaming — their function timeouts kill long SSE connections. Use Railway, Fly.io, or a VPS.

Deploy

Docker

bash 
docker build -t clinepass-bridge .
docker run -d -p 8787:8080 --name clinepass-bridge \
  -e HOST=0.0.0.0 -e PORT=8080 \
  -e API_KEYS=sk-cpb-… \
  -e CLINE_REFRESH_TOKEN=… \
  -v clinepass-cache:/app/.cache \
  clinepass-bridge

Fly.io

bash 
fly launch --no-deploy
fly secrets set \
  API_KEYS=sk-cpb-… \
  CLINE_REFRESH_TOKEN=…
fly deploy

Railway

Railway auto-detects the Dockerfile. Connect the repo, set env vars, mount a volume at /app/.cache.

🚫
Don't use Vercel or Netlify — their request timeouts kill LLM streaming. Use a VM-style host.

Troubleshooting

SymptomFix

Streaming cuts offDon't use serverless. Use a VM (Railway, Fly.io, VPS).
Port already in useChange PORT or kill existing: lsof -ti:8787 | xargs kill
401 / 403 from upstreamToken expired. Sign in with Cline CLI again.
connection refusedBridge not running. npm run dev first.
