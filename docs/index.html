ClinePass Bridge

One Pass. Every Tool.

An OpenAI-compatible API bridge that lets you use your Cline Pass subscription from any tool — Cursor, Continue.dev, Aider, LibreChat, or the openai SDK.

Introduction

Cline Pass is a subscription that gives you access to top coding models through Cline's own clients. But what if you want to use it in your tools? That's where this bridge comes in — it speaks the OpenAI API on the outside and Cline's internal protocol on the inside.

architecture 
┌────────────────────┐
│   Your favorite    │
│   OpenAI client    │  Cursor · Continue.dev · Aider · SDK
└─────────┬──────────┘
          │ OpenAI API + sk-cpb-…
          ▼
┌────────────────────┐
│  ClinePass Bridge  │  · reads ~/.cline credential store
│   (this project)   │  · auto-refreshes WorkOS tokens
└─────────┬──────────┘  · retries 401 with fresh token
          │ WorkOS bearer token
          ▼
┌────────────────────┐
│   api.cline.bot    │  kimi-k3 · glm-5.2 · deepseek-v4-pro
└────────────────────┘  qwen3.7-max · minimax-m3 · +6 more
ℹ️
How it stays fresh: the bridge reads the same credential file your Cline CLI uses. As long as you open the Cline CLI or VS Code extension occasionally, tokens rotate automatically.

Your own API keys

Generate sk-cpb-… keys per tool. Timing-safe validation, multiple keys supported.

Zero token babysitting

Auto-refresh via WorkOS. Written back to CLI store so CLI + bridge stay in sync.

Streaming built-in

Full SSE passthrough. Real-time tokens flow straight to your client.

Deploy anywhere

One codebase for local and cloud. Docker, Fly.io, Railway ready.

Quick Start

From zero to first API call in under a minute.

Clone & install
bash 
git clone https://github.com/mocasus/clinepass-bridge.git
cd clinepass-bridge
npm install
Generate your API key
bash 
npm run genkey
# → sk-cpb...xxxx
Configure environment
.env 
API_KEYS=sk-cpb...xxxx
Start the bridge
bash 
npm run dev
# → http://127.0.0.1:8787
Verify it works
bash 
curl -s http://127.0.0.1:8787/v1/chat/completions \
  -H "Authorization: Bearer sk-cpb-…" \
  -H "Content-Type: application/json" \
  -d '{"model":"kimi-k3","messages":[{"role":"user","content":"hi"}]}'
✅
Got a JSON reply with "choices"? You're live. Point any OpenAI-compatible tool at http://127.0.0.1:8787/v1.

Authentication

Two layers of auth: your client proves itself with an sk-cpb-… key, the bridge proves itself to Cline with a WorkOS token.

Token resolution order

  1. In-memory cache — fastest path, no disk I/O
  2. Bridge cache — .cache/tokens.json (self-refreshed tokens)
  3. Cline CLI store — ~/.cline/data/settings/providers.json
  4. Environment override — CLINE_ACCESS_TOKEN / CLINE_REFRESH_TOKEN
  5. WorkOS refresh — single-flight grant, persisted to both caches
🔑
When a token is refreshed, it's written back to both the bridge cache and the Cline CLI store. CLI and bridge never fight over stale tokens.

from openai import OpenAI

    base_url="http://127.0.0.1:8787/v1",

Streaming

Add "stream": true to any request. The bridge passes SSE through verbatim — no buffering, no re-chunking.

streaming.py 
from openai import OpenAI

client = OpenAI(base_url="http://127.0.0.1:8787/v1", api_key="sk-cpb-…")

stream = client.chat.completions.create(
    model="kimi-k3",
    messages=[{"role": "user", "content": "count to 10"}],
    stream=True,
)

for chunk in stream:
    print(chunk.choices[0].delta.content or "", end="")
⚠️
Serverless warning: avoid Vercel / Netlify for streaming — their function timeouts kill long SSE connections. Use Railway, Fly.io, or a VPS.

Deploy

One codebase, two modes. Local dev reads the Cline CLI store; production uses env-provided tokens.

Docker

bash 
docker build -t clinepass-bridge .
docker run -d -p 8787:8080 --name clinepass-bridge \
  -e HOST=0.0.0.0 -e PORT=8080 \
  -e API_KEYS=sk-cpb-… \
  -e CLINE_REFRESH_TOKEN=… \
  -v clinepass-cache:/app/.cache \
  clinepass-bridge

Fly.io

bash 
fly launch --no-deploy
fly secrets set \
  API_KEYS=sk-cpb-… \
  CLINE_REFRESH_TOKEN=…
fly deploy

Railway

Railway auto-detects the Dockerfile. Connect the repo, set the same env vars, and mount a persistent volume at /app/.cache so refreshed tokens survive redeploys.

🚫
Don't use Vercel or Netlify — their request timeouts kill LLM streaming. Use a VM-style host.

Troubleshooting

SymptomFix
apiKeysConfigured: 0.env not loaded. Make sure API_KEYS is set.
No refresh token availableSign in with Cline CLI once, or set CLINE_REFRESH_TOKEN.
Streaming cuts offDon't use serverless. Use a VM (Railway, Fly.io, VPS).
Port already in useChange PORT or kill existing: lsof -ti:8787 | xargs kill
401 / 403 from upstreamToken expired. Sign in with Cline CLI again.
connection refusedBridge not running. npm run dev first.
